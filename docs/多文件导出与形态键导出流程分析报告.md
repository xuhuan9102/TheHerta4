## 概述

本报告详细分析了 TheHerta4 插件中的三种导出模式：

| 导出模式 | 触发条件 | 导出轮次计算 |
|---------|---------|-------------|
| 多文件导出 | 存在 `SSMTNode_MultiFile_Export` 节点 | 物体列表长度 |
| 形态键导出 | 存在 `SSMTNode_PostProcess_ShapeKey` 节点 | 形态键数量 + 1（基态） |
| 组合导出 | 两种节点同时存在 | 形态键轮次 + 多文件轮次 |

## 组合导出流程分析

- **多文件列表数量**: 3
- **形态键数量**: 3

- **形态键名称**: `[ShapeKey_A, ShapeKey_B, ShapeKey_C]`

### 导出轮次计算

```
总导出轮次 = 形态键轮次 + 多文件轮次

           = 4 + 3
           = 7 轮
```

## 总结

### 导出模式对比

| 特性 | 多文件导出 | 形态键导出 | 组合导出 |
|-----|-----------|-----------|---------|
| 触发节点 | SSMTNode_MultiFile_Export | SSMTNode_PostProcess_ShapeKey | 两者皆有 |
| 导出轮次 | 物体列表长度 | 形态键数量 + 1 | 两者之和 |
| 文件夹命名 | Meshes01~NN | Meshes0000, Meshes1001~NNNN | 两者皆有 |
| INI生成 | 仅首轮生成 | 仅首轮生成 | 仅首轮生成 |
| 后处理 | 生成动画配置 | 生成形态键配置 | 按顺序执行两者 |

### 执行顺序

```
组合导出执行顺序:
1. 形态键导出（基态 → 各槽位）
2. 多文件导出（各帧）
3. 后处理（形态键配置 → 多文件配置）
```

### 注意事项

1. **Buffer 大小一致性检查**: 每轮导出后会检查 Buffer 文件夹大小，不一致则报错
2. **形态键值管理**: 多文件导出阶段，形态键值已归零
3. **INI 文件生成**: 仅在首轮导出生成，后续轮次仅导出 Buffer
4. **后处理顺序**: 按节点连接顺序执行，形态键配置通常先于多文件配置
