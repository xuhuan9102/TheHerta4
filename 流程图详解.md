## 图表1: 完整工作流程（导入→编辑→导出）

**说明**: 这是TheHerta4的顶层流程，展示了从游戏文件Dump到最终Mod包输出的完整过程。

### 流程要点

1. **导入阶段** (左侧，黄色背景)
   - 🎮 输入：游戏文件Dump + 工作空间
   - 📂 扫描并查找SubMesh文件夹
   - 📖 解析JSON、IB、VB文件
   - 🔄 创建Blender网格
   - 📝 保存导入记录
   - 🎨 自动生成蓝图

2. **编辑阶段** (中间，蓝色背景)
   - ✏️ 用户在Blender中手动编辑
   - 调整模型、应用形状、添加道具

3. **导出阶段** (右侧，粉色背景)
   - 🔗 解析蓝图树
   - 💾 生成缓冲文件 (IB/VB/ShapeKey)
   - 📄 生成INI配置
   - 📦 输出Mod包
   - ✅ 完成

### 游戏类型分支

图表展示了不同游戏的导出器分支：
- **EFMI/GIMI**: 原神系列
- **HIMI**: 崩坏3
- **WWMI**: 关键词模型 (⭐支持形态键)
- **Unity**: 那罗迦/少女前线2系列
- **其他**: ZZMI/SRMI等

## 图表2: 导入流程详解 (含时间成本和操作细节)

**说明**: 详细展示导入过程的每一步操作及其时间成本。

### 导入分阶段详解

| 阶段 | 代码函数 | 步骤 | 时间 | 操作 |
|-----|-------|-----|------|-----|
| **T0** | 初始化 | 0.1s | - WorkSpaceHelper.create_and_get_workspace_collection()<br/>- 扫描目录<br/>- 读取Config.json |
| **T1-T2** | 循环导入 | 主要耗时 | - 遍历SubMesh文件夹<br/>- 检查TYPE_GPU/TYPE_CPU<br/>- 读取JSON元数据<br/>- 读取二进制文件 |
| **T3** | 创建网格 | 0.5-2s | - MeshCreateHelper.create_mesh_object()<br/>- 创建网格顶点<br/>- 填充属性<br/>- 创建形态键 |
| **T4** | 记录导入 | 0.1s | - 保存Import.json<br/>- 记录SubMesh→GameType映射 |
| **T5** | 生成蓝图 | 1-3s | - 创建蓝图树<br/>- 为每个物体创建Object_Info节点<br/>- 连接到Result_Output |

### 关键决策流程

```
检查TYPE文件夹
  ├─ 优先级1: TYPE_GPU-EFMI (或其他预设)
  │   └─ 尝试导入
  │       ├─ 成功 → 记录GameType + Break
  │       └─ 失败 → 继续下一个
  │
  └─ 优先级2: TYPE_CPU-EFMI
      └─ 尝试导入
          ├─ 成功 → 记录GameType + Break
          └─ 失败 → 报错并跳过此SubMesh
```

## 图表3: 导出流程详解 (含游戏类型区分)

**说明**: 详细展示导出过程，强调不同游戏类型的区分。

### 导出关键阶段

| 阶段 | 时间 | 功能 |
|-----|------|------|
| **T30** | 0.5s | 获取蓝图树并验证 |
| **T31** | 1-3s | 解析蓝图、构建处理链 |
| **T32** | 0.1s | 选择游戏导出器 |
| **T33** | 主要耗时 | 生成缓冲文件 |
| **T34** | 1-2s | 生成INI配置文件 |

### 游戏类型导出器映射

```python
logic_name = GlobalConfig.logic_name

if logic_name == "EFMI":
    exporter = ExportEFMI()
    # 特殊处理：压缩法线解码
    
elif logic_name == "GIMI":
    exporter = ExportGIMI()
    # 特殊处理：GIMI格式优化
    
elif logic_name == "HIMI":
    exporter = ExportHIMI()
    # 特殊处理：HIMI规范适配
    
elif logic_name == "WWMI":
    exporter = ExportWWMI()
    # 特殊处理：形态键导出 ⭐
    
elif logic_name in ["Naraka", "GF2", "AILIMIT"]:
    exporter = ExportUnity()
    # 特殊处理：Unity引擎适配
    
else:
    # 其他游戏导出器...
```

### 缓冲文件生成流程

```
for each DrawIB:
  ├─ write_buf_ib_r32_uint()          → Index.buf
  ├─ write_category_buffer_files()    → Normal.buf, TexCoord.buf, ...
  └─ write_buf_shapekey_*() [WWMI]   → Position.001.buf, Position.002.buf, ...
```

## 图表5: 数据流向与文件操作

**说明**: 展示数据从输入到输出的流向过程。

### 三层数据流

```
第1层: 输入层 (工作空间文件系统)
├─ .json  (10-100 KB)   ← SubmeshJson解析
├─ .ib    (1-50 MB)     ← numpy.fromfile → R32_UINT数组
└─ .vb    (10-200 MB)   ← numpy.fromfile → 结构化数组
   │
   ↓ 读取 (numpy.fromfile + JSON解析)
   │
第2层: 处理层 (内存)
├─ NumPy数组处理
├─ Blender网格创建
├─ 蓝图处理链收集
└─ 缓冲数据重组
   │
   ↓ 写入 (struct.pack + numpy.tofile)
   │
第3层: 输出层 (Mod包)
├─ Meshes/
│  ├─ Index.buf (索引缓冲)
│  ├─ Normal.buf (法线缓冲)
│  ├─ TexCoord.buf (纹理坐标)
│  ├─ Skinning.buf (骨骼权重)
│  └─ Position.*.buf (形态键) [WWMI]
├─ Textures/
│  └─ hash*.dds (贴图)
└─ d3dx.ini (配置)
```

### 关键操作详解

| 操作 | 函数 | 参数 | 返回 |
|-----|-----|-----|------|
| **读取JSON** | JsonUtils.LoadFromFile(path) | 文件路径 | dict |
| **读取二进制** | numpy.fromfile(path, dtype, count) | 路径、类型、数量 | ndarray |
| **创建网格** | bpy.data.meshes.new(name) | 名称 | Mesh对象 |
| **添加顶点** | mesh.vertices.add(count) | 数量 | - |
| **写入顶点** | mesh.vertices.foreach_set(attr, data) | 属性、数据 | - |
| **写入文件** | struct.pack(fmt, *args) | 格式、数据 | bytes |
| **写入二进制** | numpy.tofile(file) | 文件对象 | - |

## 总结表

### 关键时间节点

| 时间点 | 事件 | 函数 | 耗时 |
|------|------|------|------|
| T0 | 初始化 | GlobalConfig.read_from_main_json_ssmt4() | 0.1s |

| T2 | 扫描目录 | WorkSpaceHelper.get_submesh_folderpath_list() | 0.1s |

### 总耗时统计

```
导入阶段:  5-30 秒

  - 网格创建:    0.5-2s
  - 蓝图生成:    1-3s

导出阶段:  10-30 秒
  - 蓝图解析:    1-3s
  - 缓冲导出:    5-20s (主要耗时)
  - INI生成:     1-2s
```

## 快速参考

### 导入快速流程

```
1. 点击导入 → ImprotFromWorkSpaceFull()
2. 扫描文件夹 → WorkSpaceHelper.get_submesh_folderpath_list()
3. 选择TYPE → TYPE_GPU优先
4. 解析JSON → SubmeshJson.__post_init__()
5. 读取缓冲 → numpy.fromfile()
6. 创建网格 → MeshCreateHelper.create_mesh_object()
7. 保存记录 → Import.json
8. 生成蓝图 → 自动创建节点树
```

### 导出快速流程

```
1. 点击生成Mod → SSMTGenerateModBlueprint.execute()
2. 获取蓝图 → BlueprintExportHelper.get_current_blueprint_tree()
3. 解析处理链 → BluePrintModel.__init__()
4. 选择导出器 → 根据logic_name
5. 导出缓冲 → BufferExportHelper.write_*()
6. 生成配置 → M_IniBuilder.save_to_file()
7. 完成 → 打开输出文件夹
```
