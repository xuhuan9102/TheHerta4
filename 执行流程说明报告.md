## 第一部分：项目与系统架构

### 1. 项目基本信息

#### 1.1 项目简介

| 项目属性 | 详细内容 |
|--------|--------|
| **项目名称** | TheHerta4 |
| **项目类型** | Blender Python 插件 (v4.5+) |
| **核心功能** | SSMT4 框架下的 3D 游戏模型 Mod 导入导出工具 |
| **所属框架** | SSMT4 (Steam Mod Manager Tools 4) |
| **开发者** | Nico (TheHerta4新架构) + 希尔 (TheHerta3维护) |
| **许可证** | 见 LICENSE.txt |
| **主仓库** | https://github.com/... (TheHerta4) |
| **分支维护** | https://github.com/xuhuan9102/TheHerta3 |

#### 1.2 支持的游戏及预设类型

TheHerta4 通过 `LogicName` 支持以下11种游戏和预设：

```
1. EFMI          - 原神 (Genshin Impact) 万叶版本预设
2. GIMI          - 原神增强改进版本
3. HIMI          - 崩坏3 (Honkai Impact 3) 预设
4. IdentityV     - 第五人格预设
5. SRMI          - 妊娠模拟器 (Simulator) 预设
6. ZZMI          - 祖玛 (ZZMI) 预设
7. WWMI          - 关键词模型 (WWMI) 预设 [支持形态键]
8. SnowBreak     - 雪碎预设
9. YYSLS         - 银河历史预设
10. Naraka       - 那罗迦预设 (Unity引擎)
11. NarakaM      - 那罗迦移动端预设
12. GF2          - 少女前线2预设 (Unity引擎)
13. AILIMIT      - AI极限预设 (Unity引擎)
```

每个游戏预设对应不同的导出模块，位于 `ui/universal/` 目录。

#### 1.3 支持的Blender版本

- **最低版本**: Blender 4.5 LTS
- **推荐版本**: Blender 4.5+, 4.4, 4.3
- **向下兼容**: 支持从 3.6 开始的版本 (需要 API 适配)
- **Python版本**: 随 Blender 版本而定 (通常 3.10+)

### 5. 文件读取完整流程

#### 5.1 读取的所有文件类型及路径

##### 5.1.1 配置文件 (Config Files)

| 文件名 | 路径变量 | 完整路径示例 | 文件格式 | 创建时机 | 职责 |
|------|--------|-----------|--------|--------|------|
| `settings.json` | `{AppData}\SSMT4\` | `%LOCALAPPDATA%\SSMT4GlobalConfigs\settings.json` | JSON | SSMT4 初始化 | 全局配置、当前工作空间、游戏选择 |
| `Config.json` (游戏) | `{SSMT}\Configs\Games\{GameName}\` | `E:\SSMT4\Configs\Games\原神\Config.json` | JSON | 游戏配置 | 游戏安装路径、游戏预设类型 |
| `Config.json` (工作空间) | `{WorkSpace}\` | `E:\SSMT4\WorkSpace\原神\char_1\Config.json` | JSON | 导入时自动生成 | DrawIB 与别名的映射关系 |
| `Import.json` (导入记录) | `{WorkSpace}\` | `E:\SSMT4\WorkSpace\原神\char_1\Import.json` | JSON | 每次导入更新 | 各 SubMesh 的 GameType 选择记录 |

**配置文件读取调用链**:
```
GlobalConfig.read_from_main_json_ssmt4()
  ├─ 打开: {AppData}\SSMT4GlobalConfigs\settings.json
  ├─ 读取: CurrentGameName, CurrentWorkSpace, SSMTWorkFolder
  ├─ 打开: {SSMT}\Configs\Games\{GameName}\Config.json
  └─ 读取: installDir, gamePreset

WorkSpaceHelper.get_drawib_aliasname_dict()
  └─ 打开: {WorkSpace}\Config.json
     └─ 读取: 各 DrawIB 对应的别名
```

##### 5.1.2 主要数据文件 (Data Files) - 导入源

| 文件类型 | 扩展名 | 路径变量 | 完整路径示例 | 二进制格式 | 大小范围 | 数量 |
|---------|-------|--------|-----------|----------|--------|------|
| JSON元数据 | `.json` | `{WorkSpace}\{DrawIB}-*\TYPE_*\` | `...\\char_1\\Type-001\\TYPE_GPU-EFMI\\Type-001.json` | JSON (UTF-8) | 10KB-100KB | 1/SubMesh |
| 索引缓冲 | `.ib` | 同上 | `...\\Type-001.ib` | 二进制 (R32_UINT) | 1MB-50MB | 1/SubMesh |
| 顶点缓冲 | `.vb` | 同上 | `...\\Type-001.vb` | 二进制 (结构化) | 10MB-200MB | 1-N/SubMesh |
| 格式定义 | `.fmt` | 同上 | `...\\Type-001.fmt` | 文本 | 1KB-10KB | 1/SubMesh |

**数据文件路径构造**:
```
base_path = GlobalConfig.path_workspace_folder()
            = "{SSMT_ROOT}\WorkSpace\{GameName}\{WorkspaceName}\"

submesh_folder = base_path + "{DrawIB}-{IndexCount}-{FirstIndex}\"
                = "...\\char_1\\Type-001-1000-0\"

data_type_folder = submesh_folder + "TYPE_{GPU|CPU}-{GameType}\"
                 = "...\\Type-001\\TYPE_GPU-EFMI\"

files:
  - json_file = data_type_folder + "{DrawIB}-{IndexCount}-{FirstIndex}.json"
  - ib_file   = data_type_folder + "{DrawIB}-{IndexCount}-{FirstIndex}.ib"
  - vb_file   = data_type_folder + "{DrawIB}-{IndexCount}-{FirstIndex}.vb"
  - fmt_file  = data_type_folder + "{DrawIB}-{IndexCount}-{FirstIndex}.fmt"
```

##### 5.1.3 文件读取调用序列 (按执行顺序)

```
阶段 1: 初始化和配置读取 (T0)
──────────────────────
1. 打开 %LOCALAPPDATA%\SSMT4GlobalConfigs\settings.json
   → GlobalConfig.gamename, workspacename, ssmtlocation
   
2. 打开 {SSMT}\Configs\Games\{GameName}\Config.json
   → GlobalConfig.current_game_migoto_folder, logic_name

阶段 2: 工作空间扫描 (T1)
──────────────────────
3. 扫描 {WorkSpace}\ 目录
   → WorkSpaceHelper.get_submesh_folderpath_list()
   → 获取所有 SubMesh 文件夹 (名称包含 '-' 的)
   
4. 打开 {WorkSpace}\Config.json (如存在)
   → WorkSpaceHelper.get_drawib_aliasname_dict()
   → 获取 DrawIB→别名映射

阶段 3: 数据导入 (T2-T5)
──────────────────────
5. 对每个 SubMesh 文件夹:
   a) 扫描 TYPE_GPU/-CPU 文件夹
      → WorkSpaceHelper.get_ordered_gpu_cpu_import_folderpath_list()
      → 优先级: TYPE_GPU 优先于 TYPE_CPU
   
   b) 打开 {SubMesh}\TYPE_*\{Name}.json
      → SubmeshJson.__post_init__()
      → JsonUtils.LoadFromFile()
      → 解析: GamePreset, CategoryBufferList, IndexBufferList
   
   c) 打开 {SubMesh}\TYPE_*\{Name}.ib
      → 二进制读取: numpy.fromfile()
      → 按 DXGI_FORMAT 转换数据类型
      → 返回: ib_data (numpy array), ib_count, ib_polygon_count
   
   d) 打开 {SubMesh}\TYPE_*\{Name}.vb (多个)
      → 对每个 CategoryBuffer:
         - 打开对应文件
         - 按定义解析顶点数据
         - 返回: vb_data (dict), vb_vertex_count
   
   e) 可选: 打开 {SubMesh}\TYPE_*\{Name}.fmt (如为 3DMigoto)
      → FMTFile 解析
      → 获取: stride, topology, format, elements

阶段 4: 记录保存 (T5)
──────────────────────
6. 生成 {WorkSpace}\Import.json
   → 记录: {submesh_name: gametype_name} 映射
   → 用于后续导出时选择正确的 TYPE 文件夹
```

### 6. 数据解析与导入具体步骤

#### 6.1 JSON 文件解析流程

**文件**: `common/submesh_json.py`  
**入口**: `SubmeshJson.__post_init__()` → `parse_json_dict()`

```python
# 步骤 1: 读取 JSON 文件
json_file_path = "{WorkSpace}\{DrawIB}-*\TYPE_*\{DrawIB}-*.json"
json_dict = JsonUtils.LoadFromFile(json_file_path)  # 返回 Python dict

# 步骤 2: 提取顶层字段
{
  "GamePreset": "EFMI",              # 游戏预设
  "VertexLimitVB": "",               # 顶点数量限制 (可选)
  "WorkGameType": "EFMI",            # 工作游戏类型
  "GPU-PreSkinning": false,          # GPU 预计算蒙皮标志
  "CategoryHash": {                  # 类别哈希映射
    "Normal": "abc123def456"
  },
  "CategoryDrawCategoryMap": {...},  # 类别和 DrawCall 映射
  "TextureMarkUpInfoList": [...],    # 纹理标记信息
  "IndexBufferList": [               # 索引缓冲列表
    {
      "FileName": "Type-001.ib",
      "DXGI_FORMAT": "R32_UINT"      # 索引格式
    }
  ],
  "CategoryBufferList": [            # 顶点缓冲列表
    {
      "FileName": "Type-001.buf",
      "Type": "Normal",              # 缓冲类型
```

```python
@staticmethod
def create_mesh_from_json(json_file_path: str, import_collection: Collection = None):
    """
    步骤流程:
    
    1. 解析 JSON 文件
       ├─ SubmeshJson(json_file_path)
       └─ 返回: submesh_json (包含所有元数据、缓冲路径等)
    
    2. 解析索引缓冲 (Index Buffer)
       ├─ parse_index_buffer(submesh_json)
       ├─ 打开: {JobType}\{DrawIBName}.ib
       ├─ 读取: numpy.fromfile(ib_path, dtype=R32_UINT)
       └─ 返回: (ib_data, ib_count, ib_polygon_count)
    
    3. 解析顶点缓冲 (Vertex Buffer)
       ├─ parse_category_buffers(submesh_json)
       ├─ for each CategoryBuffer in submesh_json.CategoryBufferList:
       │   ├─ 打开对应的 .buf 文件
       │   ├─ 按元素定义解析二进制数据
       │   └─ 收集到 vb_data dict
       └─ 返回: (elements, vb_data, vb_vertex_count)
    
    4. 创建 Blender 网格
       └─ MeshCreateHelper.create_mesh_object(
           mesh_name, source_path, logic_name, gametypename,
           elements, vb_data, ib_data, ...
          )
          └─ 返回: blender_mesh_object (创建完成的网格)
    """
    
    # 解析 JSON 文件
    submesh_json = SubmeshJson(json_file_path)
    
    # 解析索引和顶点缓冲
    elements, vb_data, vb_vertex_count = SSMTImportHelper.parse_category_buffers(submesh_json)
    ib_data, ib_count, ib_polygon_count = SSMTImportHelper.parse_index_buffer(submesh_json)
    
    # 提取元信息
    mesh_name = os.path.splitext(submesh_json.FileName)[0]
    logic_name = submesh_json.GamePreset
    gametypename = submesh_json.WorkGameType
    
    # 创建 Blender 网格
    return MeshCreateHelper.create_mesh_object(
        mesh_name = mesh_name,
        source_path = submesh_json.JsonFilePath,
        logic_name = logic_name,
        gametypename = gametypename,
        elements = elements,
        vb_data = vb_data,
        ib_data = ib_data,
        vb_vertex_count = vb_vertex_count,
        ib_count = ib_count,
        ib_polygon_count = ib_polygon_count,
        import_collection = import_collection,

# 子函数 1: 解析索引缓冲
@staticmethod
def parse_index_buffer(submesh_json: SubmeshJson):
    """
    步骤:
    1. 从 JSON 获取 IndexBuffer 信息
    2. 获取文件大小
    3. 按 DXGI_FORMAT 获取数据类型
    4. 计算索引数量 = 文件大小 / 步长
    5. 使用 numpy.fromfile 读取二进制文件
    """
    index_buffer = submesh_json.IndexBufferList[0]
    
    if not os.path.exists(index_buffer.FilePath):
        raise Fatal("Unable to find .ib file: " + index_buffer.FileName)
    
    ib_file_size = os.path.getsize(index_buffer.FilePath)
    
    index_np_type = FormatUtils.get_nptype_from_format(index_buffer.DXGI_FORMAT)
    index_stride = numpy.dtype(index_np_type).itemsize  # 通常 4 字节 (R32_UINT)
    
    ib_count = int(ib_file_size / index_stride)
    ib_polygon_count = int(ib_count / 3)  # 三角形数 = 索引数 / 3
    
    ib_data = numpy.fromfile(
        index_buffer.FilePath,
        dtype = index_np_type,
        count = ib_count
    )
    
    return ib_data, ib_count, ib_polygon_count

# 子函数 2: 解析顶点缓冲
@staticmethod
def parse_category_buffers(submesh_json: SubmeshJson):
    """
    步骤:
    1. 遍历所有 Normal 类型的 CategoryBuffer
    2. 对每个 Buffer，按照 D3D11ElementList 构建 NumPy dtype
    3. 读取二进制文件
    4. 验证顶点数量一致性
    5. 返回元素列表和数据字典
    """
    elements = []
    vb_data = {}
    vb_vertex_count = 0
    
    # 第一轮：处理 Normal 类型缓冲
    for category_buffer in submesh_json.CategoryBufferList:
        if category_buffer.Type != "Normal":
            continue
        
        # 解析此缓冲的元素和数据
        category_elements, category_vb_data, category_vertex_count = \
            SSMTImportHelper.parse_normal_category_buffer(category_buffer)
        
        # 验证和累积顶点数
        if category_vertex_count > 0:
            if vb_vertex_count == 0:
                vb_vertex_count = category_vertex_count
            elif vb_vertex_count != category_vertex_count:
                raise Fatal("Vertex count mismatch between buffers")
        
        elements.extend(category_elements)
        vb_data.update(category_vb_data)
    
    # 第二轮：处理特殊类型缓冲（如形态键等）
    for category_buffer in submesh_json.CategoryBufferList:
        if category_buffer.Type == "Normal":
            continue
        
        category_elements, category_vb_data, category_vertex_count = \
            SSMTImportHelper.parse_special_category_buffer(
                category_buffer = category_buffer,
                vb_vertex_count = vb_vertex_count,
            )
        
        elements.extend(category_elements)
        vb_data.update(category_vb_data)
    
    return elements, vb_data, vb_vertex_count
```

```python
@staticmethod
def create_mesh_object(
    mesh_name: str,
    source_path: str,
    logic_name: str,
    gametypename: str,
    elements: list,
    vb_data: dict,
    ib_data: numpy.ndarray,
    vb_vertex_count: int,
    ib_count: int,
    ib_polygon_count: int,
    import_collection: Collection = None,
) -> bpy.types.Object:
    """
    创建 Blender 网格的完整流程:
    
    ┌─────────────────────────────────────┐
    │  创建 Blender 网格对象 (6 大步骤)    │
    └─────────────────────────────────────┘
    
    步骤 1: 初始化网格数据 (Mesh)
    ├─ bpy.data.meshes.new(name=mesh_name)
    ├─ mesh.vertices.add(count=vb_vertex_count)  # 添加顶点
    └─ mesh.loops.add(count=ib_count)            # 添加环路
    
    步骤 2: 填充顶点坐标 (POSITION)
    ├─ 从 vb_data['POSITION'] 读取 float32 数组
    └─ mesh.vertices.foreach_set('co', position_data)
    
    步骤 3: 构建三角形面 (Faces/Polygons)
    ├─ mesh.polygons.add(count=ib_polygon_count)  # 添加面
    ├─ 从 ib_data 构建面索引
    └─ mesh.polygons.foreach_set('loop_indices_start', ...)
    
    步骤 4: 设置顶点属性
    ├─ 颜色 (COLOR):
    │   └─ mesh.color_attributes.new(name='COLOR', type='FLOAT_COLOR', domain='POINT')
    │
    ├─ 法线 (NORMAL):
    │   └─ 解码或导入法线数据
    │
    ├─ 纹理坐标 (TEXCOORD):
    │   └─ mesh.uv_layers.new(name=f'uv_{index}')
    │       └─ uv_layer.data.foreach_set('uv', uv_data)
    │
    └─ 骨骼权重 (BLENDWEIGHTS + BLENDINDICES):
        ├─ 创建顶点组 (Vertex Groups)
        └─ foreach_set(weights)
    
    步骤 5: 创建形态键 (ShapeKeys)
    ├─ 检测 SHAPEKEY 类型元素
    ├─ mesh.shape_keys.new(name='Basis')  # 基础形态键
    └─ for each shapekey:
        ├─ shape_block = mesh.shape_keys.key_blocks.new(name)
        └─ shape_block.data.foreach_set('co', offset_data)
    
    步骤 6: 创建对象并链接到场景
    ├─ obj = bpy.data.objects.new(name=mesh_name, object_data=mesh)
    ├─ collection.objects.link(obj)
    └─ 设置自定义属性:
        ├─ obj['source_path'] = source_path
        ├─ obj['logic_name'] = logic_name
        ├─ obj['gametypename'] = gametypename
        └─ obj['submesh_name'] = mesh_name
    """
    
    # === 步骤 1 & 2: 创建网格和设置顶点坐标 ===
    mesh = bpy.data.meshes.new(name=mesh_name)
    mesh.vertices.add(count=vb_vertex_count)
    mesh.loops.add(count=ib_count)
    
    # 从 vb_data 中提取 POSITION
    position_key = next((k for k in vb_data.keys() if 'POSITION' in k), None)
    if position_key:
        position_data = vb_data[position_key].flatten()
        mesh.vertices.foreach_set('co', position_data)
    
    # === 步骤 3: 构建三角形面 ===
    mesh.polygons.add(count=ib_polygon_count)
    mesh.polygons.foreach_set('loop_indices_start', ib_data)
    
    mesh.update()  # 更新网格
    
    # === 步骤 4: 设置顶点属性 ===
    
    # 设置颜色
    color_key = next((k for k in vb_data.keys() if 'COLOR' in k), None)
    if color_key:
        color_attr = mesh.color_attributes.new(name='COLOR', type='FLOAT_COLOR', domain='POINT')
        color_attr.data.foreach_set('color', vb_data[color_key].flatten())
    
    # 设置 UV
    texcoord_keys = [k for k in vb_data.keys() if 'TEXCOORD' in k]
    for i, texcoord_key in enumerate(texcoord_keys):
        uv_layer = mesh.uv_layers.new(name=f'uv_{i}')
        uv_data = vb_data[texcoord_key].flatten()
        uv_layer.data.foreach_set('uv', uv_data)
    
    # === 步骤 5: 创建形态键（如存在） ===
    shapekey_keys = [k for k in vb_data.keys() if 'SHAPEKEY' in k]
    if shapekey_keys:
        # 创建基础形态键
        mesh.shape_keys.new(name='Basis')
        
        for shapekey_key in shapekey_keys:
            shape_block = mesh.shape_keys.key_blocks.new(name=f'Shape_{shapekey_key}')
            shape_block.data.foreach_set('co', vb_data[shapekey_key].flatten())
    
    # === 步骤 6: 创建对象并链接到场景 ===
    obj = bpy.data.objects.new(name=mesh_name, object_data=mesh)
    
    if import_collection:
        import_collection.objects.link(obj)
    else:
        bpy.context.scene.collection.objects.link(obj)
    
    # 设置自定义属性
    obj['source_path'] = source_path
    obj['logic_name'] = logic_name
    obj['gametypename'] = gametypename
    obj['submesh_name'] = mesh_name
    
    return obj
```

```python
def execute(self, context):
    """
    执行时间序列（按调用顺序）:
    
    T30 - 获取和验证蓝图 (0.5s)
    ├─ BlueprintExportHelper.get_current_blueprint_tree(context)
    │   └─ 从 context 获取当前活跃的蓝图树
    │
    └─ BlueprintExportHelper.set_runtime_blueprint_tree(tree)
        └─ 将蓝图树存储为运行时全局变量
    
    T31 - 解析蓝图树 (1-3s)
    └─ BluePrintModel(tree=tree, context=context)
        ├─ 遍历蓝图节点树
        ├─ 为每个物体追踪处理链 (ProcessingChain)
        ├─ 收集形态键、重命名规则等
        └─ 返回: blueprint_model (包含 ordered_draw_obj_data_model_list)
    
    T32 - 选择导出器 (0.1s)
    ├─ 根据 GlobalConfig.logic_name 选择对应导出器
    │   ├─ EFMI     → ExportEFMI
    │   ├─ GIMI     → ExportGIMI
    │   ├─ HIMI     → ExportHIMI
    │   ├─ Unity    → ExportUnity (Naraka/GF2/AILIMIT)
    │   ├─ IdentityV → ExportIdentityV
    │   ├─ SRMI     → ExportSRMI
    │   ├─ ZZMI     → ExportZZMI
    │   ├─ WWMI     → ExportWWMI
    │   ├─ SnowBreak → ExportSnowBreak
    │   └─ YYSLS    → ExportYYSLS
    │
    └─ exporter = ExportXXX(blueprint_model=blueprint_model)
    
    T33 - 执行导出 (10-30s)
    └─ exporter.export()
        ├─ 遍历每个 DrawIB
        │
        ├─ generate_buffer_files()
        │   ├─ 导出索引缓冲 (Index.buf)
        │   ├─ 导出顶点缓冲 (Category.buf)
        │   └─ 导出形态键缓冲 (Position.*.buf)
        │
        └─ build_mod_files_dict() + save_to_file()
            ├─ 生成 INI 配置文件
            └─ 写入文件系统
    
    T34 - 完成
    └─ return {'FINISHED'}
    """
    
    # ============ 实现代码 ============
    
    # 步骤 1: 获取和验证蓝图
    tree = BlueprintExportHelper.get_current_blueprint_tree(context)
    if not tree:
        self.report({'ERROR'}, "未找到当前蓝图")
        return {'CANCELLED'}
    
    BlueprintExportHelper.set_runtime_blueprint_tree(tree)
    
    # 步骤 2: 解析蓝图
    try:
        blueprint_model = BluePrintModel(tree=tree, context=context)
    except ValueError as error:
        self.report({'ERROR'}, str(error))
        return {'CANCELLED'}
    
    LOG.info(f"蓝图解析完成: {len(blueprint_model.ordered_draw_obj_data_model_list)} 个物体")
    
    # 步骤 3: 根据游戏类型选择导出器
    if GlobalConfig.logic_name == LogicName.EFMI:
        exporter = ExportEFMI(blueprint_model=blueprint_model)
    elif GlobalConfig.logic_name == LogicName.GIMI:
        exporter = ExportGIMI(blueprint_model=blueprint_model)
    elif GlobalConfig.logic_name == LogicName.HIMI:
        exporter = ExportHIMI(blueprint_model=blueprint_model)
    elif GlobalConfig.logic_name == LogicName.Naraka or GlobalConfig.logic_name == LogicName.GF2:
        exporter = ExportUnity(blueprint_model=blueprint_model)
    # ... 其余游戏类型 ...
    else:
```

#### 8.2 蓝图解析（BluePrintModel）

**类**: `blueprint/model.py::BluePrintModel`  
**职责**: 解析蓝图树，生成处理链

```python
class BluePrintModel:
    """
    蓝图模型 - 解析蓝图树并追踪每个物体的完整处理链
    
    核心概念:
    ├─ ProcessingChain: 物体从 Object_Info 到 Result_Output 的完整路径
    ├─ 节点参数签名: 用户在蓝图中设置的参数
    └─ 处理链合并: 相同参数的物体可以共享一个处理链
    """
    
    def __init__(self, tree: bpy.types.NodeTree, context):
        """
        初始化和解析流程:
        
        1. 查找 Result_Output 节点（必须存在且唯一）
        2. 从 Result_Output 反向追踪所有连接的源
        3. 对每个找到的源（Object_Info 或 Object_Group）:
           ├─ 构建 ProcessingChain
           ├─ 追踪中间处理节点 (ShapeKey, Rename等)
           └─ 收集参数签名
        4. 按 DrawIB 分组组织物体
        5. 生成 ordered_draw_obj_data_model_list (导出顺序)
        """
        
        self.tree = tree
        self.context = context
        
        # 找到 Result_Output 节点
        result_output_node = None
        for node in tree.nodes:
            if node.bl_idname == 'SSMTNode_Result_Output':
                result_output_node = node
                break
        
        if not result_output_node:
            raise ValueError("未找到 Result_Output 节点")
        
        # 从 Result_Output 反向追踪源
        self.processing_chains = []
        self._trace_from_result_output(result_output_node)
        
        # 按 DrawIB 分组
        self.ordered_draw_obj_data_model_list = self._group_by_draw_ib()
    
    def _trace_from_result_output(self, output_node):
        """
        从 Result_Output 反向追踪所有源
        
        蓝图连接示例:
        ┌──────────────────┐
        │ Object_Info      │
        │  (object_1)      │
        └──────┬───────────┘
               │ link
        ┌──────▼───────────┐
        │ ShapeKey         │
        │  (shape_1)       │
        └──────┬───────────┘
               │ link
        ┌──────▼───────────┐
        │ Result_Output    │
        │  (output)        │
        └──────────────────┘
        
        追踪过程:
        1. output_node.inputs[0] 得到输入 socket
        2. socket.links[0].from_node 得到前一个节点 (ShapeKey)
        3. 递归处理前一个节点
        """
        
        # 获取 Result_Output 的输入 socket
        if len(output_node.inputs) == 0:
            return
        
        input_socket = output_node.inputs[0]  # 通常是第一个输入
        
        for link in input_socket.links:
            from_node = link.from_node
            
            if from_node.bl_idname == 'SSMTNode_Object_Info':
                # 找到源节点
                self._create_processing_chain(from_node, [from_node], output_node)
            
            elif from_node.bl_idname == 'SSMTNode_Object_Group':
                # GROUP 节点作为中继
                self._trace_through_group(from_node, output_node)
            
            else:
                # 可能是处理节点，继续反向追踪
                self._trace_backwards(from_node, [from_node], output_node)
    
    def _create_processing_chain(self, source_node, node_path, output_node):
        """
        为单个物体创建处理链
        
        ProcessingChain 数据结构:
        {
          object_name: "Type-001.自定义名称",
          source_node: <Object_Info node>,
          node_path: [Object_Info, ShapeKey, Rename, ...],
          node_param_signatures: [
            "ObjectInfo[obj=Type-001...]",
            "ShapeKey[name=shape1,vk=...]",
            "Rename[...]"
          ],
          reached_output: True,
          is_valid: True
        }
        """
        
        object_name = source_node.object_name
        
        # 提取节点的参数签名
        node_param_sigs = []
        for node in node_path:
            sig = ProcessingChain.extract_node_signature(node)
            node_param_sigs.append(sig)
        
        # 创建处理链
        chain = ProcessingChain(
            object_name = object_name,
            original_object_name = object_name,
            source_node = source_node,
            node_path = node_path,
            node_param_signatures = node_param_sigs,
            reached_output = True,
            is_valid = True
        )
        
        self.processing_chains.append(chain)
    
    def _trace_backwards(self, current_node, node_path, output_node):
        """
        从处理节点向前追踪，直到找到 Object_Info
        """
        
        if len(current_node.inputs) == 0:
            return
        
        for input_socket in current_node.inputs:
            for link in input_socket.links:
                from_node = link.from_node
                new_path = [from_node] + node_path
                
                if from_node.bl_idname == 'SSMTNode_Object_Info':
                    self._create_processing_chain(from_node, new_path, output_node)
                else:
                    self._trace_backwards(from_node, new_path, output_node)
    
    def _group_by_draw_ib(self):
        """
        按 DrawIB 分组组织物体
        
        返回: {
          DrawIB-1: [chain1, chain2, ...],
          DrawIB-2: [chain3, chain4, ...],
          ...
        }
        """
        
        draw_ib_dict = {}
        
        for chain in self.processing_chains:
            # 从物体名称或蓝图属性提取 DrawIB
            draw_ib = chain.source_node.draw_ib
            
            if draw_ib not in draw_ib_dict:
                draw_ib_dict[draw_ib] = []
            
            draw_ib_dict[draw_ib].append(chain)
        
        return draw_ib_dict
```

#### 8.3 缓冲导出

**函数**: `common/buffer_export_helper.py::BufferExportHelper.write_*()`

```python
class BufferExportHelper:
    """
    缓冲区导出工具 - 将 Blender 数据写入二进制文件
    
    输出位置:
    {GameInstallDir}\Mods\{WorkspaceName}\Meshes\
    
    输出文件类型:
    1. Index.buf      - 索引缓冲 (R32_UINT)
    2. {Category}.buf - 顶点缓冲 (结构化二进制)
    3. Position.*.buf - 形态键缓冲 (float16 或 float32)
    """
    
    @staticmethod
    def write_buf_ib_r32_uint(index_list: list[int], buf_file_name: str):
        """
        导出索引缓冲为 R32_UINT 格式
        
        参数:
            index_list: 整数索引列表 [0, 1, 2, 1, 2, 3, ...]
            buf_file_name: 输出文件名 (如 "Type-001-Index.buf")
        
        过程:
        1. 构建输出文件路径 = {buffer_folder} + buf_file_name
        2. 将索引列表按 uint32 big-endian 打包
        3. 写入二进制文件
        """
        
        ib_path = os.path.join(GlobalConfig.path_generatemod_buffer_folder(), buf_file_name)
        
        # 使用 struct.pack 打包为小端 (Little-Endian) uint32
        packed_data = struct.pack(f'<{len(index_list)}I', *index_list)
        
        with open(ib_path, 'wb') as f:
            f.write(packed_data)
        
        print(f"已导出索引缓冲: {ib_path} (大小: {len(packed_data)/1024/1024:.2f}MB)")
    
    @staticmethod
    def write_category_buffer_files(category_buffer_dict: dict, draw_ib: str):
        """
        导出顶点缓冲（各类别）
        
        参数:
            category_buffer_dict: {
              'Normal': numpy.ndarray,      # 法线缓冲
              'TexCoord': numpy.ndarray,    # 纹理坐标缓冲
              'Skinning': numpy.ndarray,    # 骨骼权重缓冲
              ...
            }
            draw_ib: DrawIB 标识 (如 "Type-001")
        
        过程:
        1. 遍历每个类别缓冲
        2. 构建文件名 = "{draw_ib}-{category_name}.buf"
        3. 使用 numpy.ndarray.tofile() 写入
        """
        
        for category_name, category_buf in category_buffer_dict.items():
            buf_path = os.path.join(
                GlobalConfig.path_generatemod_buffer_folder(),
                f"{draw_ib}-{category_name}.buf"
            )
            
            with open(buf_path, 'wb') as f:
                category_buf.tofile(f)
            
            print(f"已导出顶点缓冲: {buf_path}")
    
    @staticmethod
    def write_buf_shapekey_offsets(shapekey_offsets: list, filename: str):
        """
        导出形态键偏移数据 (int32)
        
        用于指定每个形态键的顶点偏移
        """
        
        with open(os.path.join(GlobalConfig.path_generatemod_buffer_folder(), filename), 'wb') as f:
            for number in shapekey_offsets:
                data = struct.pack('i', number)  # int32
                f.write(data)
    
    @staticmethod
    def write_buf_shapekey_vertex_offsets(shapekey_vertex_offsets: list, filename: str):
        """
        导出形态键顶点偏移 (float16)
        
        用于 WWMI 等支持形态键的导出器
        """
        
        float_array = numpy.array(shapekey_vertex_offsets, dtype=numpy.float32)
        float_array = float_array.astype(numpy.float16)  # 转换为 float16 以节省空间
        
        with open(os.path.join(GlobalConfig.path_generatemod_buffer_folder(), filename), 'wb') as f:
            float_array.tofile(f)
```

```python
class M_IniBuilder:
    """
    INI 配置文件生成器
    
    生成游戏 Mod 配置文件 (d3dx.ini)
    
    INI 结构:
    ├─ [TextureOverride] 部分 (纹理替换)

    ├─ [Resource] 部分 (资源定义)
    └─ [VSChecksum] 部分 (着色器校验)
    """
    
    def __init__(self):
        self.line_list = []  # 生成的所有行
        self.ini_section_list = []  # Section 对象列表
        self.ini_section_name_set = set()  # 已添加过的 Section 名称
    
    def append_section(self, m_inisection):
        """
        添加一个 Section
        
        参数:
            m_inisection: M_IniSection 对象
        
        过程:
        1. 检查 Section 是否为空
        2. 如果不为空，添加到列表
        """
        
        if not m_inisection.empty():
            self.ini_section_list.append(m_inisection)
    
    def build_line_list(self):
        """
        生成完整的 INI 文件行列表
        
        过程:
        1. 按 Section 类型遍历
        2. 对每个 Section，添加 [SectionName] 头
        3. 添加 Section 的所有行
        4. 自动去重（相同名称的 Section 仅出现一次）
        """
        
        # 定义 Section 类型的显示顺序
        section_order = [
            'Present',
            'Constants',
            'TextureOverrideIB',
            'TextureOverrideVB',
            'TextureOverrideVertexLimitRaise',
            'TextureOverrideTexture',
            'TextureOverrideGeneral',
            'CustomShader',
            'ResourceBuffer',
            'ResourceTexture',
            'VertexShaderCheck',
        ]
        
        for section_type in section_order:
            self.__append_section_line(section_type)
    
    def save_to_file(self, file_path: str):
        """
        保存 INI 文件到磁盘
        
        参数:
            file_path: 输出文件路径 (如 "{game_mods_folder}\d3dx.ini")
        
        过程:
        1. 构建完整的行列表
        2. 打开文件进行写入
        3. 逐行写入
        """
        
        self.build_line_list()
        
        with open(file_path, 'w', encoding='utf-8') as f:
            for line in self.line_list:
                f.write(line)
        
        print(f"已生成 INI 文件: {file_path}")
```

#### 9.2 INI Section 类型和职责

```
INI 结构分层:

第 1 层: 基础配置 (通常在文件顶部)
├─ [Present]        # 显示设置 (呈现参数)

├─ [CustomShader] - Base            # 着色器基类
├─ [CustomShader] - Depth          # 深度着色器
└─ [CustomShader] - Other          # 其他着色器

第 4 层: 资源定义 (Resource 系列)
├─ [Resource]Buffer               # 缓冲资源定义
├─ [Resource]Texture             # 纹理资源定义
└─ [Resource]ShapeKeys           # 形态键资源 (WWMI)

└─ [VSChecksum]                  # 顶点着色器校验和
```

### 12. 数据格式转换参考表

#### 12.1 二进制格式对应的 NumPy 类型

| DXGI 格式 | D3D11 类型 | NumPy 类型 | 字节数 | 用途 |
|---------|---------|---------|------|------|
| `R32_UINT` | 无符号32位整数 | `numpy.uint32` | 4 | 索引缓冲 |
| `R32G32B32_FLOAT` | 浮点向量3 | `(float32, 3)` | 12 | 顶点坐标/法线 |
| `R32G32_FLOAT` | 浮点向量2 | `(float32, 2)` | 8 | 纹理坐标 |
| `R8G8B8A8_UNORM` | 8位RGBA | `(uint8, 4)` | 4 | 顶点颜色 |
| `R16G16_SNORM` | 有符号16位向量2 | `(int16, 2)` | 4 | 压缩法线 |
| `R16G16B16A16_FLOAT` | 浮点向量4 | `(float16, 4)` | 8 | 权重数据 |

#### 12.2 顶点数据 Stride 计算

```
总 Stride = 所有元素 ByteWidth 之和

示例:
  POSITION (R32G32B32_FLOAT) = 12字节
  TEXCOORD (R32G32_FLOAT)    = 8字节
  COLOR    (R8G8B8A8_UNORM)  = 4字节
  ──────────────────────────
  总 Stride = 12 + 8 + 4 = 24字节

验证: 顶点缓冲文件大小 % Stride = 0 (必须整除)
```
